## Karpathy Coding Principles (always active)
- **Think Before Coding**: State assumptions, present alternatives, push back on bad approaches, stop when confused
- **Simplicity First**: Minimum code solving the problem, no speculative features or premature abstractions
- **Surgical Changes**: Touch only what's needed, match existing style, don't drive-by refactor
- **Goal-Driven Execution**: Define success criteria, verify with tests, loop until confirmed
- **Anti-patterns**: No over-abstraction, no speculative features, no drive-by refactoring, no silent assumptions
- Full skill: `/a0/skills/karpathy-guidelines/SKILL.md`

## Deslop Writing Rules (always active)
- Never use banned words: delve, tapestry, leverage, seamlessly, groundbreaking, transformative, pivotal, harness, robust, scalable, holistic, synergy, empower, streamline, foster, facilitate, utilize, comprehensive
- Never use filler phrases: it's worth noting, I cannot stress enough, notably, certainly, in conclusion, in summary, needless to say, it goes without saying, great question, of course
- No "It isn't just X, it's Y" constructions
- No em-dash overuse, no question-then-immediate-answer hooks
- Start with substance, not framing. End with impact, not summary
- Active voice + strong verbs. Vary sentence length deliberately
- Specific > generic. Concrete numbers over vague quantifiers
- Full skill + banned word list: `/a0/skills/deslop/SKILL.md`
- Cross-ref: `python3 scripts/cross-reference.py build`
