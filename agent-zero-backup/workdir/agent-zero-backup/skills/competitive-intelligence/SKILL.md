# Competitive Intelligence Skill

## When to Use
Activate when asked to compare tools, products, services, or protocols. Use for market research, feature comparisons, and competitive positioning.

## The Process

### Step 1: Define the Landscape
- What category are we analyzing?
- Who are the key players?
- What dimensions matter to the user?

### Step 2: Feature Matrix

```markdown
## Feature Comparison: [Category]

| Feature | [Product A] | [Product B] | [Product C] |
|---------|-------------|-------------|-------------| |
| [Feature 1] | Yes | No | Yes |
| [Feature 2] | Yes | Yes | Partial |
| Pricing | [Price] | [Price] | [Price] |
| [Feature N] | [Status] | [Status] | [Status] |
```

### Step 3: SWOT Analysis (per competitor)

```markdown
## [Product Name] SWOT

**Strengths**
- [Key advantage]

**Weaknesses**
- [Key limitation]

**Opportunities**
- [Market gap they could exploit]

**Threats**
- [External risk]
```

### Step 4: Positioning Map
Describe where each player sits on key axes:
- Price: Budget to Premium
- Complexity: Simple to Complex
- Target: Individual to Enterprise
- Openness: Open to Proprietary

### Step 5: Recommendation
Based on the analysis, provide:
- Best for [use case A]: [Product]
- Best for [use case B]: [Product]
- Best overall: [Product]

## Constraints
- Base comparisons on verifiable features, not opinions
- Include pricing when available
- Acknowledge when data is incomplete
- Update comparisons when versions change
