# SCQA Writing Framework

## When to Use
Activate when asked to write or structure content using the SCQA framework — threads, essays, pitch docs, presentations, or any communication that needs logical flow.

## The SCQA Process

### Step 1: S — Situation
Set the context. Explain the current state or background so the audience understands the starting point.
- What is the current reality?
- What does the audience already know?
- Be factual and concise.

### Step 2: C — Complication
Introduce a problem, tension, or challenge that disrupts the situation. This is what makes the story or argument interesting.
- What changed or went wrong?
- What's the gap between the current state and the desired state?
- Create urgency or curiosity.

### Step 3: Q — Question
Pose the key question that naturally arises from the complication. What does the audience want answered?
- Should be a single, clear question.
- Must flow logically from the complication.
- Frame it to lead into your answer.

### Step 4: A — Answer
Provide the solution, insight, or recommendation. This resolves the question logically.
- Be specific and actionable.
- Support with evidence or examples.
- End with a clear takeaway or call-to-action.

## Output Format

```markdown
## [Title]

**Situation:** [Context and background]

**Complication:** [The problem or tension]

**Question:** [The key question]

**Answer:** [The solution with supporting points]
```

## Tips
- Keep each section focused — one idea per block
- The Complication is the most important part — it creates engagement
- Use this for consulting reports, executive summaries, and persuasive content
- Think of it as storytelling with logic
