03

The DOT Matrix = BRQC Level 2 (Metrics Building)

BRQC defines Level 2 as metrics building — the DOT matrix is this artifact

  • Uses 7 Business Optimizations as the taxonomy for value
  • Uses 6 Data Processes to structure the lifecycle of each metric
  • Forces every Tier metric to justify: which optimization it serves, which data process it relies on, and what decision it enables
Verbatim: "The DOT matrix prevents metric sprawl by requiring explicit value justification." This is exactly how BRQC describes optimizations — as an evaluation lens, not a list of KPIs.
04

How Tier Metrics Are Designed (BRQC-Aligned)

Metrics written as decision signals, not activity reports

  • Metrics are written as objectives to minimize, maximize, or eliminate
  • Metrics describe decision signals, not activities
  • Persistence and aggregation matter more than point values
BRQC alignment: This mirrors BRQC's emphasis on observable behaviors over outcome slogans (e.g., "reduce NCs" is not a metric). Supports BRQC's stance on avoiding Goodhart's Law in quality metrics.
